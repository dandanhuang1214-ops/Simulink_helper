# Demo 交付验收清单

## 一、启动与持久化

- 使用 `scripts\start-demo.ps1` 启动GPU模式。
- `scripts\check-demo.ps1` 中Web、API、Ollama、Qdrant、SQLite均正常。
- `docker exec simulink-assistant-ollama ollama ps` 在模型运行期间显示GPU处理器。
- 刷新网页、重启API和重启整套容器后，会话、文档、Wiki与向量数据不丢失。
- 停止服务时不得使用 `docker compose down -v`。

## 二、固定问答题

| 类型 | 问题 | 验收重点 |
|---|---|---|
| 闲聊路由 | 你好，你能做什么？ | 快速直答，不检索知识库 |
| 定义 | Stateflow是什么？ | 定义准确，引用Stateflow资料 |
| 关系 | Simulink和AUTOSAR是什么关系？ | 覆盖建模、ARXML、仿真或代码生成 |
| 对比 | 固定步长和可变步长求解器有什么区别？ | 证据覆盖两类求解器，不混用无关证据 |
| 测试 | MIL测试与Simulink Test有什么联系？ | 命中测试领域资料并给出引用 |
| 无答案 | 当前知识库没有覆盖的非相关专业问题 | 明确证据不足，不编造引用 |

知识回答至少检查：正文完整、引用可点击、右栏证据匹配、PDF页码可定位、后台Judge不阻塞答案显示。

## 三、聊天交互

- Enter发送，Shift+Enter换行。
- 生成期间停止按钮可用；中断后保留部分答案并可重试。
- 复制、重新生成、赞、踩和查看证据可用。
- 切换会话不会串流；刷新后不存在永久“正在生成”。
- 历史会话可搜索、置顶、重命名和删除。
- 长回答可以自由滚动，并可点击“回到底部”。

## 四、知识与图谱

- 文档上传、任务状态、失败原因和重建入口可用。
- Wiki页面引用能够打开对应证据。
- 图谱搜索输入流畅，滚轮缩放与拖动无明显阻塞。
- 图谱搜索结果保留匹配节点标签；节点点击可查看一跳关系。

## 五、Demo边界

- 当前为单用户本地Demo，不包含登录、RBAC和公网部署。
- 扫描PDF、复杂表格、图片语义和公式识别不作为本版强制验收项。
- 本地2B Judge只用于观察，不作为生产级质量结论。
- 图谱是Wiki、实体和证据的派生索引，不修改原始知识资产。
