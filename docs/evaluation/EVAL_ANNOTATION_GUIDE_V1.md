# Evaluation Annotation Guide v1

这份规范用于把当前 25 题“预评测”升级成可计算指标的正式评测集。

核心原则：不要为了让当前答案过关而标注。标注对象是“理想系统应该如何回答”，不是“当前系统刚好回答了什么”。

## 标注对象

每个问题需要标注：

| 字段 | 说明 |
|---|---|
| `decision` | `pass` / `partial` / `fail` / `revise_question` / `remove` |
| `expected_mode` | `rag` / `refusal` / `direct` |
| `expected_evidence` | 期望命中的证据 chunk id，可填多个 |
| `must_cover` | 答案必须覆盖的关键点 |
| `forbidden` | 答案不能出现的内容，例如脑补 UI 步骤、无证据结论 |
| `answer_score` | 0-2 分：0 错误，1 部分可用，2 可接受 |
| `faithfulness_score` | 0-2 分：0 编造，1 部分证据支撑，2 基本都被证据支撑 |
| `citation_score` | 0-2 分：0 无引用/错引用，1 引用不完整，2 引用覆盖关键结论 |
| `notes` | 人工备注 |

## 判定标准

### pass

问题质量合格，当前答案也基本可接受。允许存在轻微表达问题，但不能有关键事实错误。

### partial

问题值得保留，但当前答案存在明显不足，例如：

- 回答过泛；
- 引用覆盖不足；
- 混入弱相关证据；
- 有轻微脑补但不影响主结论；
- 答案结构可读性差。

### fail

当前答案不可接受，例如：

- 核心事实错误；
- 把无证据内容说成事实；
- 应拒答却回答；
- 应回答却拒答；
- 引用与结论明显不对应。

### revise_question

问题本身需要改写，例如：

- 问得太大；
- 混合了多个独立问题；
- 当前知识库不足以支撑，但又不适合作为边界题；
- 缺少版本/资料范围限制。

### remove

当前阶段不适合放入评测集。

## 指标设计

人工标注完成后，可以计算以下指标：

| 指标 | 计算方式 | 作用 |
|---|---|---|
| Recall@K | 期望证据是否出现在 Top-K 候选中 | 衡量检索召回 |
| MRR | 第一个期望证据的排名倒数 | 衡量正确证据排位 |
| Evidence Precision | 最终引用中有多少是期望/相关证据 | 衡量引用干净度 |
| Answer Accuracy | 人工答案分是否达标 | 衡量答案是否正确 |
| Faithfulness | 答案结论是否被证据支撑 | 衡量是否编造 |
| Citation Coverage | 关键结论是否都有引用 | 衡量引用覆盖 |
| Refusal Accuracy | 边界题是否正确拒答 | 衡量安全边界 |

## 当前优先人工审核的问题

优先审核这些题，因为它们最容易暴露链路问题：

1. Q010：AUTOSAR 代码生成 vs 普通 C/C++，容易回答过泛或证据不足。
2. Q011：AUTOSAR XML 导入流程，容易脑补 UI 步骤。
3. Q016：Stateflow state/transition，容易被弱相关表格证据带偏。
4. Q017-Q019：多跳题，容易混入跨主题弱证据。
5. Q020-Q024：边界题，检查拒答是否足够清楚。

## 标注建议

- `expected_evidence` 不要求一次填全，可以先填 1-3 个“核心必命中”证据。
- 如果当前答案引用了不相关证据，把它写入 `forbidden` 或 `notes`。
- 如果问题需要更精确，优先标 `revise_question`，不要硬给当前问题打低分。
- 对于边界题，`expected_evidence` 可以为空，重点标注 `expected_mode=refusal` 和拒答理由。
